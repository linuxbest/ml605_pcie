Change log for axi_dma



 

Changes in 6.00.a introduced in 14.1

14.1 - Changes in VHDL/Verilog/Netlist sources (.vhd, .v, .ngc, .edn) 

New Features:

 

  • Multi Channel DMA support enabled. Channel selection is based on TDEST. Refer UG for more information.

 

  • A number of new registers have been added for S2MM path. Refer the user guide for more information.
  • Enabled Keyhole read/write feature.
  • Store and Forward is now an optional feature when streaming data width is same as Mmap data width.

 

Resolved Issues:

 

 

Known Issues / Limitations:

  • Axi_dma_v6_00_a is fully backward compatible. To move your existing designs to the new version make sure that parameters

C_ENABLE_MULTI_CHANNEL is set to 0, C_NUM_MM2S_CHANNELS is set to 1 and C_NUM_S2MM_CHANNELS is set to 1.

                   

 

 

Changes in 5.00.a introduced in 13.3

13.3 - Changes in VHDL/Verilog/Netlist sources (.vhd, .v, .ngc, .edn) 

New Features:

 

  • DMA now uses axi_datamover_v4_00_a and axi_sg_v4_00_a.

 

  • New feature, Store and Forward is added (C_INCLUDE_MM2S_SF, C_INCLUDE_S2MM_SF). This feature helps in reducing the bubbles in the AXI transaction. This feature is enabled inside the RTL.

 

Resolved Issues:

 

 

Known Issues / Limitations:

  • When migrating from older version you may want to take into consideration the new parameters.



 

Changes in 4.00.a introduced in 13.2

13.2 - Changes in VHDL/Verilog/Netlist sources (.vhd, .v, .ngc, .edn) 

New Features:

 

  • Made Scatter Gather Engine optional. When the Scatter Gather engine is excluded then Simple DMA mode is enabled.

 

  • CR589995 - Added TCL script process to detect asynchronous clock mode and set/clear C_PRMRY_IS_ACLK_ASYNC automatically.

 

  • CR585409 - Optimized AXI4 Stream ports to remove resets on data buses. This allows for easier FPGA placement and timing closure.

 

Resolved Issues:

 

  • CR587591 - Renamed AXI Control and Status Stream bus interfaces in mpd to M_AXIS_MM2S_CNTRL and S_AXIS_S2MM_STS to allow automatic tool connection of AXI Monitor.

 

 

Known Issues / Limitations:

  • No known issues or limitations




 

Changes in 3.00.a introduced in 13.1

13.1 - Changes in VHDL/Verilog/Netlist sources (.vhd, .v, .ngc, .edn) 

New Features:

 

  • Added option to clock all memory map interfaces asynchronously from eachother. This modified clock interface to AXI DMA giving four clock inputs, s_axi_lite_aclk, m_axi_sg_aclk, m_axi_mm2s_aclk, and m_axi_s2mm_aclk.

 

  • CR582801 Updated AXI Stream data width parameter to ***_TDATA_WIDTH to allow tools to autoconnect AXI Stream Monitor.

 

Resolved Issues:

 

  • CR578807 - Renamed AXI Stream ports from *_tstrb to *_tkeep

 

 

Known Issues / Limitations:

  • No known issues or limitations






 

Changes in 2.00.a introduced in 12.4

12.4 - Changes in VHDL/Verilog/Netlist sources (.vhd, .v, .ngc, .edn) 

New Features:

 

  • No New Features

 

Resolved Issues:

 

  • CR573702 - Renamed m_axis_mm2s_aresetn,m_axis_mm2s_cntrl_aresetn, s_axis_s2mm_aresetn, and s_axis_s2mm_sts_aresetn to be non-AXI based.

 

  • CR573642 - Added C_INTERCONNECT_M_AXI_MM2S_READ_ISSUING=4 and C_INTERCONNECT_M_AXI_S2MM_WRITE_ISSUING=4 to mpd and mui to improve AXI DMA performance.

 

  • CR574977 - Removed Frequency limit from input clocks in MPD file.

 

  • CR576999 - Fixed issue with back to back read and write request on AXI Lite interface. Subsequent requests were ignored.

 

  • CR576336 - Modified MPD and MUI file BUS_STD and added PROTOCOL parameters for AXI Stream interface to support new tool directive.
