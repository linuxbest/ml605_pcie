Changes in 3.01.a introduced in 14.1

14.1 - Changes in VHDL/Verilog/Netlist sources (.vhd, .v, .ngc, .edn)

 
New Features:
  • Updated with CoreGen core Virtex-6 Embedded Tri-Mode Ethernet Mac Wrapper v2.2 for Virtex-6 hard TEMAC implementations.
  • Updated with CoreGen core Tri-Mode Ethernet Mac v5.3 for soft TEMAC implementations.
  • CoreGen core Ethernet AVB Endpoint v3.1, now part of Tri-Mode Ethernet MAC v5.3.
  • Updated with CoreGen core Ethernet 1000BASE-X PCS/PMA or SGMII v11.3.
  • Supports AXI_DMA_v5_00_a
  • Added support for Kintex7, Virtex-7, both GTX and GTH, and Artix-7(with GTP) for all PHY features.
  • Axi_Ethernet validated on KC705 Rev C board for all features.
  • Support and constraints added and tested for Vivado for series-7 families.
Resolved Issues:
  • None
Known  Issues / Limitations:
  • [< CR626500 >] .to be fixed in 14.2

14.1 - Changes in tool interface files (.mpd)

  • Added C_USE_GTH parameter for Virtex-7 constraints.
  • Updated tcl constraints for 7 series family.
  • C_AVB =1 with C_TYPE =2 is no longer supported.

 

Changes in 3.00.a introduced in 13.2.0

13.2.0 - Changes in VHDL/Verilog/Netlist sources (.vhd, .v, .ngc, .edn)

 
New Features:
  • Updated with CoreGen core Virtex-6 Embedded Tri-Mode Ethernet Mac Wrapper v2.1 for Virtex-6 hard TEMAC implementations.
  • Updated with CoreGen core Tri-Mode Ethernet Mac v5.1 for soft TEMAC implementations.
  • Updated with CoreGen core Ethernet AVB Endpoint v3.1.
  • Updated with CoreGen core Ethernet 1000BASE-X PCS/PMA or SGMII v11.1.
  • Supports AXI_DMA_v4_00_a
  • Supports AXI_FIFO_MM_S_v2_00_a
  • Added support for Virtex-7 and Kintex-7 device families
Resolved Issues:
  • None
Known  Issues / Limitations:
  • [< CR609798 >] Transmit Flow Control - The pause value may me requested such that the timing is just right for the upper 8-bits of the previous pause value to be used with the lower 8-bits of a subsequent pause value in the transmitted pause frame.
  • Support is not available for Virtex-7 and Kintex-7 SGMII and 1000BASE-X PHY interfaces
  • Support for RGMII v1.3 has been removed
  • Ability Register does not return the correct value.

13.2.0 - Changes in tool interface files (.mpd)

  • Added call to tcl for automatic constraint generation
  • Removed support for C_PHY_TYPE = 2 since RGMII v1.3 is no longer supported
  • Added Parameters for C_STATS_WIDTH and C_AXI_STR_TXC_ACLK_FREQ_HZ which are set automatically
  • Removed ISVALID modifier for GTX_CLK input so that it is always needed since it is now used to generate PHY_RST_N output
  • Made C_BASEADDR and C_HIGHADDR parameters NON_HDL since they are only used by the tools and not using or present inside the core design

13.2.0 - Changes in Tcl script files associated with core (.tcl)

  • Implemented vast majority of constraints to .tcl with exception of IODELAY and IOB constraints.
  • Removed some commented out unused code

13.2.0 - Changes in documentation associated with core

  • Minor updates for version change

 

Changes in 2.01.a introduced in 13.1.0

13.1.0 - Changes in VHDL/Verilog/Netlist sources (.vhd, .v, .ngc, .edn)

 
New Features:
  • Updated with CoreGen core Virtex-6 Embedded Tri-Mode Ethernet Mac Wrapper v1.4 for Virtex-6 hard TEMAC implementations.
  • Updated with CoreGen core Tri-Mode Ethernet Mac v4.4 for soft TEMAC implementations.
  • Updated with CoreGen core Ethernet AVB Endpoint v2.4.
  • Updated with CoreGen core Ethernet 1000BASE-X PCS/PMA or SGMII v10.5.
  • Updated with CoreGen core Ethernet Statistics v3.4.
  • Supports AXI_DMA_v3_00_a
  • Supports AXI_FIFO_MM_S_v2_00_a
Resolved Issues:
  • Renamed AXIStream _TSTRB port signal names to _TKEEP. This more accurately conforms to the AXI standard. There is no change to function or use. This must be used with DMA and FIFO cores with similar port signal name changes.
  • Enabled full checksum offload
  • Updated to newer axi_lite_ipif core to reduce resources.
  • [< CR575688 >] 12.4 - axi_ethernet_v1_01_a, As requested, modified axi_ethernet_soft_temac_wrap and axi_ethernet designs to default the receive operation to disabled following reset. This eliminated possible coordination issues between axi_ethernet, the other AXIStream cores (axi_dma or axi_fifo_mm_s), and the software drivers and application when resetting while connected to a live network.The core design will now require a write to register offset 0x404 setting bit 28 to enable the RX interface before the core will work.
Known  Issues / Limitations:
  • None

13.1.0 - Changes in tool interface files (.mpd)

  • Renamed AXIStream _TSTRB port signal names to _TKEEP. This more accurately conforms to the AXI standard. There is no change to function or use. This must be used with DMA and FIFO cores with similar port signal name changes.
  • [< CR588359 >] 12.4 - The AXI bus BUS_STD and PROTOCOL parameters were modified in v1_01_a for 12.4 to new values. These correspond to changes in the axi_dma and axi_fifo_mm_s cores which must match. axi_ethernet_v2_01_a requires axi_dma_v3_00_a or axi_fifo_mm_s_v2_00_a for AXI-S interface compatibility. [<AR39930>] can be referenced for more details.

Changes in 1.01.a introduced in 12.4.0

12.4.0 - Changes in VHDL/Verilog/Netlist sources (.vhd, .v, .ngc, .edn)

 
New Features:
  • None
Resolved Issues:
  • [< CR574422 >] [<AR38412>] 12.3 - axi_ethernet_v1_00_a, Statistics counter operation is incorrect when using Virtex-6 hard TEMAC running at 10/100 Mbps. Update to latest CoreGen statistics core.
  • [< CR574422 >] [<AR38601>] 12.3 - axi_ethernet_v1_00_a, Ethernet AVB operation is incorrect when running at 100 Mbps. Updated to latest AXI Stream to Client Interface shim code.
  • [<AR?????>] 12.3 - axi_ethernet_v1_00_a, Ethernet AVB TX PTP frame values and RX VLAN priority field values are based on older AVB standard. Updated to latest CoreGen AVB core.
  • [< CR574720 >] 12.3 - axi_ethernet_v1_00_a, Removed unnecessary clock frequency range restriction on AXI bus clock.
  • [< CR575229 >] 12.3 - axi_ethernet_v1_00_a, Fixed a TX bug with RGMII and SGMII when operating at 10/100 Mbps.
  • [< CR578548 >] 12.3 - axi_ethernet_v1_00_a, When using a Virtex 6 hard TEMAC , C_TYPE=2, and the C_INCLUDEIO parameter = 0, an RX clock was not connected.
  • 12.3 - axi_ethernet_v1_00_a, When using a soft TEMAC, C_TYPE = 1, and either SGMII or 1000BASE-X, the MgtRdy interrupt was connected high rather than being connected to the MGT ready output. Connected interrupt correctly.
  • 12.3 - axi_ethernet_v1_00_a, When using a soft TEMAC in 10/100 Mbps only mode, C_TYPE = 0, the power-up values of some registers do not match the data sheet. Corrected power-up register values.
Known  Issues / Limitations:
  • None
